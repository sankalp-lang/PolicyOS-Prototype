# PolicyOS · Tara — deploy & connect

Static site. No build step. Open `index.html` or serve the folder.

## Deploy with git (static host)

```bash
cd PolicyOS-Prototype
git init && git add -A && git commit -m "PolicyOS · Tara prototype"
# create an empty repo on GitHub, then:
git remote add origin git@github.com:<you>/policyos-tara.git
git push -u origin main
```

Then point any static host at the repo root (no framework, no build command):
- **Vercel** → New Project → Framework preset: **Other**, Output dir: `.`
- **Netlify** → New site from Git → Build command: *(none)*, Publish dir: `.`
- **GitHub Pages** → Settings → Pages → deploy from branch root

## Make it live (after deploy)

Nothing is committed with keys — you add them at runtime in the app:

1. **Connect a model** (top-right of the Ask-Tara panel, the landing "Connect model" button, or Connectors → *Tara's model*). Pick any provider — Gemini, ChatGPT, Claude, Sarvam, Grok, Perplexity — paste that provider's API key, optionally add a **fallback** model. With no key, Tara runs the offline demo engine.
2. **Connect sources** (Connectors page). Keka, greytHR, Jira, Notion (and the rest) each connect via **API key**, **MCP server URL**, or **OAuth**. Paste your real credentials.
3. Ask Tara — answers are generated by your model, grounded only in the data the signed-in user is permitted to see.

## How keys are handled
- Keys live **only in the browser** (`localStorage`) and are sent **only** to the provider/connector you choose. Nothing is logged or proxied through us.
- Anthropic browser calls use `anthropic-dangerous-direct-browser-access: true`. Gemini uses `x-goog-api-key`. OpenAI / Grok / Perplexity / Sarvam use their OpenAI-compatible `/chat/completions`.
- **Production hardening:** for a real deployment, move model + connector calls behind a small server-side proxy so keys never touch the client, and replace the demo RBAC with your IdP + each source's real ACLs. The permission-faithful pattern (filter the context to the user *before* the model sees it) stays the same.

## Verify endpoints before going live
Provider REST shapes occasionally change. The Anthropic and OpenAI shapes are current; Gemini / Sarvam / Grok / Perplexity are implemented to their documented OpenAI-compatible / REST endpoints — confirm with one real call per provider you intend to use.
